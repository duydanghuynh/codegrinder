Getting started on this assignment

You are now ready to access this problem from the grind tool. If you already have it set up, go to your project directory and use the following command to download this problem set:

grind get

Setting up the grind tool for the first time, or upgrading it

Click on the name of your operating system to open the instructions:

Linux

Follow these steps to set up CodeGrinder. To upgrade grind, you only need to repeat the 1st step. If you need to log in again, you only need to repeat the 3rd step:

  1. Type the following commands:
    mkdir -p ~/bin
    cd ~/bin
    curl --compressed /grind.linux -o grind
    chmod 755 grind
    cd -
  2. Log out and log back in. Type grind and hit enter. If it works, skip this step. If it complains that it cannot find the grind command, add the following line to .profile in your home directory:
    export PATH=$PATH:$HOME/bin
    Then log out and log back in, or run this command to make it take effect in the same session:
    . ~/.profile
  3. Login with grind by typing:
    grind login

Windows

Note: if you are using Windows 10, install Bash for Windows and then use the Linux instructions from a Bash shell. This is simpler and works much better than the normal Windows method.

Follow these steps to set up CodeGrinder. To upgrade grind, you only need to repeat the 3rd step. If you need to log in again, you only need to repeat the 4th step:

  1. Make a folder on your U: drive called winbin (or pick your own location and remember it)
  2. Add this directory to your Path using the Control Panel. Search online for help if you do not know how to do this
  3. Download the grind.exe tool by right-clicking here and selecting “Save link as” or “Save target as” to save it to that directory. If the security system tries to scare you away from doing this, stand up for yourself and do it anyway
  4. From a Powershell window, login with grind by typing:
    grind login
